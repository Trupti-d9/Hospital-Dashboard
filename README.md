## What This Is

A full Business Intelligence Dashboard for the CEO of Aarogya Multi-Specialty Hospital, Chennai. One screen. All departments. AI-powered. Voice-enabled. No PowerPoint needed — the dashboard explains itself.

## Quick Start (Artifact / Prototype)

The `HospitalCEO_Dashboard.jsx` is a self-contained React component that runs immediately in Claude.ai artifacts or any React environment.

**To run in Claude.ai:** Paste the JSX content into a new artifact.

**To run locally:**
```bash
npx create-react-app hospital-dashboard
cd hospital-dashboard
# Replace src/App.js content with HospitalCEO_Dashboard.jsx
npm start
```

**To run the full stack (Locally - No Docker):**
```bash
git clone <repo>
# 1. Start the Backend (FastAPI + SQLite + Neo4j + ChromaDB)
cd backend
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload
# API Docs: http://localhost:8000/docs

# 2. Start the Frontend
cd ../frontend
npm install
npm run dev
# Frontend: http://localhost:3000
```

## Architecture Overview

```
CEO Browser
    │
    ├─ React Dashboard (Tab-based SPA)
    │       ├─ Overview / Finance / Patients / Departments
    │       ├─ HR & Payroll / OT / Insurance / Appointments
    │       └─ AI Concierge (RAG + Voice)
    │
    ├─ FastAPI Backend
    │       ├─ /api/overview, /api/finance, /api/patients
    │       ├─ /api/hr, /api/ot, /api/insurance, /api/appointments
    │       ├─ /api/alerts  ← rule engine
    │       └─ /api/ai/chat ← RAG + OpenRouter API
    │
    ├─ Modular Database & Security (DPDP Compliant)
    │       ├─ SQLite (SQLCipher Encrypted): Relational records
    │       ├─ Neo4j: Graph relations for advanced analytics
    │       ├─ MFA (TOTP) enforcement for CEO login
    │       └─ (seed_data.py generates synthetic rows into both DBs)
    │
    └─ RAG Pipeline
            ├─ ChromaDB vector store
            ├─ LlamaIndex query engine
            └─ OpenRouter API (Claude 3.5 Sonnet / Llama 3)
```

## AI Concierge Capabilities

1. **Natural Language Q&A** — Ask "Which dept is most profitable?" or "What's our OT utilization?"
2. **RAG-backed** — Answers grounded in actual hospital DB, not hallucinated
3. **Voice Input** — Click 🎤, speak in English (Indian accent supported via `lang=en-IN`)
4. **Quick Prompts** — One-tap chips for common CEO questions
5. **Fallback** — If API unavailable, local intent-matching engine responds
6. **Alerts integration** — Concierge proactively surfaces critical alerts

## Predictive Alerts Logic

Alerts are generated by a rule engine with these thresholds:

| Metric | Warning | Critical |
|--------|---------|----------|
| Bed Occupancy | >80% | >90% |
| Insurance TAT | >21 days | >30 days |
| OT Cancellation Rate | >10% | >15% |
| Revenue MoM drop | >3% | >5% |
| Staff Attrition | >5% | >8% |

## Indian Hospital KPI Glossary

- **OPD** — Outpatient Department
- **IPD** — Inpatient Department
- **TPA** — Third Party Administrator (insurance intermediary)
- **BOR** — Bed Occupancy Rate
- **LOS / ALOS** — (Average) Length of Stay
- **TAT** — Turnaround Time
- **OT** — Operation Theatre
- **NABH** — National Accreditation Board for Hospitals
- **CGHS** — Central Government Health Scheme
